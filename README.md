# 🏥 MedSafe AI — Real-Time Healthcare Monitoring System

> An AI-powered real-time patient monitoring system that streams vital signs via Apache Kafka, detects anomalies using deep learning, and alerts medical staff instantly.

---

## 📌 Overview


**MedSafe AI** analyzes patient vital signs in real time using a dual-model AI pipeline — an **Autoencoder** (deep learning) combined with an **Isolation Forest** (unsupervised ML) — to detect abnormal health conditions automatically.

Patient data flows through **Apache Kafka**, is classified by severity, stored in a database, and visualized on a **live Streamlit dashboard**. Critical cases trigger **email alerts** to notify healthcare staff without delay.

---

## 🚀 Features

| Feature | Description |
|---|---|
| ⚡ Real-time streaming | Patient vitals streamed via Apache Kafka |
| 🧠 AI anomaly detection | Autoencoder + Isolation Forest dual-model pipeline |
| 🎯 Severity classification | LOW / MEDIUM / HIGH scoring system |
| 📊 Live dashboard | Streamlit-based monitoring UI |
| 📧 Email alert system | Instant alerts for HIGH severity cases |
| 🗄️ Data persistence | SQLAlchemy + PostgreSQL storage |
| 🔌 REST API | Flask API to query latest vitals and patient history |
| 📈 Scalable architecture | Decoupled producer/consumer design |

## 🔌 Apache Kafka Setup

### Download Kafka

Get the latest binary from: https://kafka.apache.org/downloads

Extract to `C:\kafka` (Windows) or `/opt/kafka` (Linux/macOS).

### Start Kafka (KRaft Mode — no Zookeeper needed)

**Windows (PowerShell):**
```powershell
cd C:\kafka
.\bin\windows\kafka-server-start.bat .\config\server.properties
```

**Linux / macOS:**
```bash
cd /opt/kafka
bin/kafka-server-start.sh config/server.properties
```

### Create the Kafka Topic

**Windows:**
```powershell
.\bin\windows\kafka-topics.bat --create --topic patient_vitals --bootstrap-server localhost:9092
```

**Linux / macOS:**
```bash
bin/kafka-topics.sh --create --topic patient_vitals --bootstrap-server localhost:9092
```

## ▶️ Running the Project

Open **four separate terminals** and run each component:

### Terminal 1 — Start Kafka
```bash
# Windows
cd C:\kafka && .\bin\windows\kafka-server-start.bat .\config\server.properties

# Linux/macOS
cd /opt/kafka && bin/kafka-server-start.sh config/server.properties
```

### Terminal 2 — Run Data Producer
```bash
python producer.py
```
Generates random patient vitals and streams them to Kafka every 2 seconds.

**Example output:**
```
Sent: {'patient_id': 'P3', 'heart_rate': 110, 'temperature': 38.5, 'blood_pressure': 160}
```

### Terminal 3 — Run AI Consumer
```bash
python consumer.py
```
Reads from Kafka, runs anomaly detection, stores results, and sends alerts if needed.

**Example output:**
```
Processed: {'patient_id': 'P4', 'heart_rate': 125, 'temperature': 39.1, 'blood_pressure': 170, 'anomaly_score': 0.72, 'severity': 'HIGH'}
```

### Terminal 4 — Run Dashboard
```bash
streamlit run dashboard.py
```

Open in browser: **http://localhost:8501**
